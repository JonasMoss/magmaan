# ML convergence diagnostics

## Problem

The SEM-zoo benchmark smoke (`benchmarks/r/run_benchmark.R`) found magmaan's ML
estimator failing on two cases that lavaan fits cleanly:

- `bollen_democracy_sem` — `LineSearchFailed` after 0 iterations.
- `bfi_5factor` — `OptimizerNonConvergence` (iteration cap reached).

## Method

A C++ diagnostic harness, `tests/unit/convergence_diagnostics_test.cpp`, probes
the objective, gradient, optimizer, and conditioning at the points that matter.
It reads the shared lavaan-parity fixtures
(`tests/fixtures/parity/<case>/reference.json`, generated by
`tools/regen_parity_fixtures.R`) which carry the sample covariance, `n_obs`, the
model syntax, and lavaan's converged θ̂ plus lavaan's own start values — the
latter two mapped into magmaan's free-parameter order. The pass/fail regression
gate now lives in `tests/golden/lavaan_parity_golden_test.cpp`; this harness
keeps the staged diagnostic report.

Note on conventions: magmaan's `$fmin` is the full ML discrepancy `F`; lavaan
reports `F/2`. So magmaan's `F` at the optimum equals `2 × lavaan fmin`.

## Hypothesis verdicts

- **H1 — gradient correctness: CLEARED.** Analytic vs central-difference
  gradients agree to < 1e-8 at magmaan's start values and at lavaan's θ̂, for
  every parameter class (loadings, residual variances/covariances, latent
  variances/covariances, regressions).
- **H6 — constraint handling: CLEARED.** Neither model has linear constraints;
  the equality reparam is the identity. Started at lavaan's θ̂, magmaan reaches
  lavaan's exact `fmin`, confirming the model and reparam are correct.
- **H5 — conditioning: secondary.** The FD-Hessian condition number at θ̂ is
  ~8e3 (bollen) / ~2e3 (bfi) — moderate. bfi converges in 1–5 iterations once
  near the optimum, so conditioning is not the primary cause.

Two root causes remain.

### A. The L-BFGS line search fails at the optimum; magmaan reports a hard error

magmaan's unbounded L-BFGS (`src/optim/lbfgs_optimizer.cpp`, LBFGS++) uses a
Nocedal-Wright strong-Wolfe line search and an **absolute** gradient-norm stop,
`gtol = 1e-7`. The optimizer descends to the optimum, but the strong-Wolfe line
search cannot satisfy its conditions in the flat neighbourhood of the minimum
before `‖g‖` reaches 1e-7 — the objective no longer changes measurably along
any step. LBFGS++ throws; the `catch` in `lbfgs_optimizer.cpp` converts it to
`FitError::LineSearchFailed` and **discards the converged iterate held in `x`**.

Evidence:

- bollen, heuristic start: `gtol=1e-7` → `LineSearchFailed` with `f=0.508336`
  (exactly the optimum, `2 × 0.254168`); `gtol=1e-5` → converges in 204
  iterations to the same `f`, θ accurate to 1.3e-4.
- bfi, started at lavaan's θ̂: `gtol=1e-7` → `LineSearchFailed`; `gtol≥1e-5` →
  converges in 1–5 iterations.
- Ceres trust-region (no line search) converges from bollen's heuristic start
  in 111 iterations where L-BFGS fails outright.

### B. `simple_start_values` is too crude for larger, sign-mixed models

`src/estimate/start_values.cpp::simple_start_values` starts every free loading
at `+0.7` regardless of sign. bfi has reverse-keyed items, so roughly half its
true loadings are large and negative (θ̂ ≈ −1.5 … −2.0). From the
`+0.7`-everywhere start neither L-BFGS (objective crawls 2.17 → 2.12 over
1k → 20k iterations) nor Ceres reaches the optimum. From lavaan's start values
magmaan *does* reach the bfi optimum — and then hits root cause A.

So bollen needs only fix A; bfi needs both.

## Fix

- **A — `src/optim/lbfgs_optimizer.cpp`:** on a line-search exception, evaluate
  the gradient at the last iterate `x`; if it is a converged stationary point
  (small gradient norm) return it as success instead of erroring. This changes
  nothing for runs that currently succeed — it only salvages the error path.
- **B — `src/estimate/start_values.cpp`:** sign each free loading's `0.7`
  start by the covariance with the factor's marker indicator, so reverse-keyed
  indicators start in the correct orthant instead of a uniform `+0.7`. The
  magnitude stays `0.7`, keeping equality-constrained loadings consistent.

## Verification

After the fix: `bollen_democracy_sem` and `bfi_5factor` converge from the
default heuristic start and match lavaan within tolerance
(`benchmarks/r/run_benchmark.R`); the existing `tests/fixtures/fit/*` stay
green; the diagnostic harness and new regression tests pass.
